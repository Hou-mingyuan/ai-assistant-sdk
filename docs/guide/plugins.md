# 前端插件系统

前端插件系统允许宿主项目向小助手界面注册自定义按钮和动作。

## 基本用法

```ts
registerPlugin({
  id: 'open-help',
  label: '帮助',
  icon: '?',
  position: 'header',
  action: async (context) => {
    context.appendUserMessage('请说明当前页面可以做什么')
  },
})
```

## 常见位置

| 位置 | 用途 |
| --- | --- |
| `header` | 顶部工具按钮，例如帮助、设置、反馈。 |
| `footer` | 输入区附近动作，例如插入模板、上传附件。 |
| `context-menu` | 消息右键菜单，例如复制、导出、从此处分叉。 |

## 插件设计建议

- 插件 `id` 应全局唯一。
- 插件动作要尽量短，不要阻塞界面。
- 涉及网络请求时应自行处理失败提示。
- 不要在插件里直接操作组件内部 DOM，优先使用公开上下文能力。

## 适合插件化的能力

- 业务系统跳转。
- 快捷 Prompt。
- 当前页面数据采集。
- 审批、工单、知识库等宿主系统动作。
